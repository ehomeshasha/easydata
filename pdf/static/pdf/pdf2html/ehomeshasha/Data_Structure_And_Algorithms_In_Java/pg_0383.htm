find()
increments
hashVal
and goes back to the top of the
while
loop to check if the
next cell is occupied. Here's the code for
find()
:
public DataItem find(int key) // find item with key
// (assumes table not full)
{
int hashVal = hashFunc(key); // hash the key
while(hashArray[hashVal] != null) // until empty cell,
{ // found the key.
if(hashArray[hashVal].iData == key)
return hashArray[hashVal]; // yes, return item
++hashVal; // go to next cell
hashVal %= arraySize; // wrap around if necessary
}
return null; // can't find item
}
As
hashVal
steps through the array, it eventually reaches the end. When this happens
we want it to wrap around to the beginning. We could check for this with an
if
statement,
setting
hashVal
to 0 whenever it equaled the array size. However, we can accomplish
the same thing by applying the
%
operator to
hashVal
and the array size.
Cautious programmers might not want to assume the table is not full, as is done here.
The table should not be allowed to become full, but if it did, this method would loop
forever. For simplicity we don't check for this situation.
The
insert()
Method
The
insert()
method uses about the same algorithm as
find()
to locate where a
data item should go. However, it's looking for an empty cell or a deleted item (key ¨C1),
rather than a specific item. Once this empty cell has been located,
insert()
places the
new item into it.
public void insert(DataItem item) // insert a DataItem
// (assumes table not full)
{
int key = item.iData; // extract key
int hashVal = hashFunc(key); // hash the key
// until empty cell or -1,
while(hashArray[hashVal] != null &&
hashArray[hashVal].iData != -1)
{
++hashVal; // go to next cell
hashVal %= arraySize; // wrap around if necessary
}
hashArray[hashVal] = item; // insert item
} // end insert()
The
delete()
Method
The
delete()
method finds an existing item using code similar to
find()
. Once the
item is found,
delete()
writes over it with the special data item
nonItem
, which is
predefined with a key of ¨C1.
